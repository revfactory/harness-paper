# Survey Synthesis — 서베이 리포트 종합 절차

서베이의 가치는 **검증된 사실을 응집된 지형으로 엮되, 커버리지 한계를 정직하게 밝히는 것**이다. 빈 곳을 그럴듯한 가짜로 채우면 서베이 전체가 무가치해진다.

## 절대 원칙 — 검증 게이트

`_workspace/03_verification/verdicts.md`에서 **CONFIRMED인 논문만 본문에 인용**한다.
- OUT-OF-RANGE / REJECTED / UNVERIFIED → 본문 배제, **부록 A에 사유와 함께 투명 기록**
- 이 게이트를 우회해 미검증 논문을 본문에 넣는 것은 금지. 서베이 신뢰성의 유일한 방어선이다.

## 없는 것을 지어내지 않는다

해당 기간(2026-05~07) 확인 문헌이 빈약하면 그 사실을 규모와 함께 밝힌다: "확인된 관련 논문은 N편이며, 그중 M편이 범위 내"처럼. 다음은 금지:
- 존재하지 않는 논문을 본문에 인용
- CONFIRMED 근거 없이 "많은 연구가 ~"라고 일반화
- 빈약한 문헌을 감추기 위한 추상적 서술로 분량 채우기

## 택소노미는 데이터에서 귀납한다

미리 정한 카테고리에 억지로 끼우지 말고, 실제 CONFIRMED 논문들의 공통 주제에서 분류를 만든다. 참고 축(강제 아님):
- 컨텍스트 엔지니어링 / 툴 오케스트레이션 / 에이전트 메모리·장기 과제 / 멀티에이전트 오케스트레이션 / 평가·벤치마크 하네스 / 코딩 에이전트 하네스

논문이 2편이면 2개 축으로 충분하다. 축 개수를 문헌 규모에 맞춘다.

## 트렌드·발전 방향은 근거를 동반한다

- 트렌드 주장마다 근거 논문을 인용한다. 단일 논문에서 성급히 일반화하지 않는다.
- 발전 방향은 논문들의 "한계" 섹션에서 도출한다. 저자·분석자가 지적한 미해결 과제를 종합한다.

## 인용은 추적 가능하게

모든 주장에 `[저자, YYYY-MM, "제목", URL]`을 붙인다. 독자가 클릭해 원문을 확인할 수 있어야 한다.

## 리포트 구조

최종 출력: `/Users/robin/Downloads/harness-paper/harness-engineering-survey-2026.md`

```markdown
# Harness Engineering 서베이: 2026년 5~7월 최신 연구 동향

## 요약 (Executive Summary)
## 1. 서론 — 하네스 엔지니어링이란
## 2. 조사 방법과 커버리지        ← 검색 전략·소스·기간·확인/검증 편수·한계 정직하게
## 3. 택소노미 — 연구 지형
## 4. 논문별 상세 분석             ← 주제 그룹별. 각 논문: 링크·핵심기여·방법·결과·한계
## 5. 관통하는 트렌드              ← 각 트렌드마다 근거 논문 인용
## 6. 발전 방향과 미해결 과제
## 7. 결론
## 부록 A: 검증 실패·범위 밖 항목 (투명성)
## 부록 B: 전체 인용 목록          ← 모든 CONFIRMED 논문 완전 서지
```

## 에러 핸들링

- CONFIRMED가 매우 적음: 규모를 명시하고, OUT-OF-RANGE 항목으로 맥락 보강(범위 내/밖 명확 구분). 부풀리지 않는다.
- 분석 간 상충(같은 논문 다른 해석): 삭제하지 않고 양측 병기 + 출처.
- 복수 주제에 걸친 논문: 주 주제에 배치하고 교차 언급.
